# Commercial Licensing

MeetingMinutesTaker is released under the **GNU Affero General Public License v3.0 (AGPL-3.0)** — see [LICENSE](LICENSE).

AGPL-3.0 is a strong copyleft license. It is a good fit for:

- Personal, research, and non-commercial use
- Internal tools at organizations that are willing to publish the full source of their modifications (including any network-accessible modifications) under AGPL-3.0

AGPL-3.0 is **not** a good fit for:

- Incorporating this code into a closed-source commercial product
- Offering this code (or a modified version of it) as a hosted service ("SaaS") without releasing your modifications under AGPL-3.0
- Embedding this code in proprietary software distributed to customers
- Any use where AGPL-3.0's source-disclosure obligations conflict with your business or contractual constraints

For these cases, a **commercial license** is available from the copyright holder on a per-engagement basis.

## What a commercial license grants

A commercial license is a private, negotiated agreement that replaces AGPL-3.0 for the licensee. Typical terms include:

- Permission to use, modify, and distribute the software **without** AGPL-3.0's source-disclosure obligations
- Scope defined per engagement (product, company, user count, deployment model)
- Optional support, SLAs, and/or maintenance terms
- Pricing agreed upfront

## Frequently asked

**Q: Can I evaluate the software before purchasing a commercial license?**
Yes. AGPL-3.0 already permits free use for evaluation. You only need a commercial license when you start using the software in a way AGPL-3.0 does not permit.

**Q: Do contributors to the repository need a commercial license?**
No. Contributions to this repository remain under AGPL-3.0. Contributors sign a [Contributor License Agreement](CLA.md) that grants the copyright holder the right to offer the contributed code under both AGPL-3.0 and a commercial license.

**Q: Does my company's internal, non-distributed modification need a commercial license?**
AGPL-3.0 triggers source-disclosure whenever the software is used to **interact with users over a network** or distributed. If your modification runs purely as an internal tool with no external network users, AGPL-3.0 may be acceptable. When in doubt, contact us.
